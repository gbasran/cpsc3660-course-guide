MODULE 1: Foundations

Core concepts, the entity-relationship model, and the relational model itself. Everything you need before touching queries or design theory.

01Database Concepts
Schema vs instance (filing cabinet analogy). DBMS components. Data dictionary. Keys (super, candidate, primary; ID-card analogy). Integrity constraints. Cascading deletes. Mirrors the prof's opening 5-6 concept questions (2m each, always on every exam).
 02E-R Model & Diagrams
Entities (strong/weak), attributes (simple, composite, multivalued, derived), relationships, cardinality, participation, weak entities with discriminators, generalization/ISA, ER→relational mapping rules. Family-tree analogy. Mirrors A3 Q4 and Q8 (bookstore with generalization).
 03Relational Model
Tuples, attributes, domains, schemas, primary keys, foreign keys, referential integrity, NULL handling. Spreadsheet analogy. The formal language your SQL and RA will operate on.
MODULE 2: Query Languages

The two query languages the prof tests heavily: Relational Algebra and SQL. The final will have ~25-30 marks of SQL and ~10-12 marks of RA.

04Relational Algebra
Every operator: σ select, π project, × product, ⋈ natural join, ∪ − ∩ set ops, ρ rename, ÷ division, γ grouping. Deep dive on division; it is the hard pattern that keeps returning (A1 Q7b, A2 Q1b; 85% probability on final).
 05SQL: DDL + Basics
CREATE TABLE with PK/FK, SELECT/FROM/WHERE, JOIN, ORDER BY, DISTINCT, BETWEEN, LIKE, IN. "Don't over-join" discipline per the prof's Week 14 hint. Mirrors A2 Q6-Q7 and Midterm Q7-Q8d.
 06SQL: Advanced
Aggregate functions, GROUP BY + HAVING (the 95%-guaranteed pattern), views with update semantics, correlated subqueries, NOT EXISTS, ALL/ANY/SOME, INTERSECT/EXCEPT, NULL handling. Mirrors A3 Q1-Q3 and Midterm Q6-Q9.
 07RA to SQL Translation
Side-by-side RA→SQL translation for every operator. Special attention to division→NOT EXISTS+EXCEPT (the hard one). Mirrors A2 Q3-Q5.
MODULE 3: Design Theory

The theoretical backbone. These topics were 0 marks on the midterm but will be ~40-45% of the final. Heaviest weight: normalization (15m BCNF showpiece).

08Functional Dependencies
FD definition; Armstrong's 3 core axioms (reflexivity, augmentation, transitivity) and 3 derived rules (union, decomposition, pseudo-transitivity); attribute closure algorithm; F⁺ closure. Full pseudo-transitivity proof (A4 Q4; high reappearance probability). Family-tree-of-rules analogy.
 09Normalization & BCNF
Heaviest page on the guide. 1NF→2NF→3NF→BCNF ladder. Candidate key enumeration. BCNF violation detection. BCNF decomposition algorithm walked verbatim on A4 Q6. Lossless test and dependency-preservation test with the two Week 14 handwritten examples (Borrow-loan lossless; Employee/Project/Branch lossy). Jigsaw-puzzle analogy.
MODULE 4: Safety Net
